1. Executive Summary
Bags Signal adalah platform intelijen token berbasis AI yang dibangun di atas ekosistem Bags.fm dan jaringan Solana. Platform ini dirancang untuk memenangkan Bags.fm Hackathon dengan menyediakan utilitas nyata bagi trader, kreator token, dan developer yang berinteraksi dengan Bags App Store.

Platform ini menggabungkan tiga pilar utama yang menjadi fokus penilaian hackathon:
• Fee Share Intelligence: Kalkulator dan optimizer real-time untuk memaksimalkan pendapatan fee sharing bagi creator dan partner.
• AI Signal Engine: Sistem scoring berbasis machine learning untuk menilai potensi setiap token launch di Bags.fm berdasarkan on-chain data, liquidity depth, dan social signals.
• One-Click Token Launcher: Interface terintegrasi untuk launch token + konfigurasi fee share + listing Dexscreener dalam satu alur transaksi.

2. Problem Statement
2.1 Latar Belakang
Bags.fm telah muncul sebagai salah satu launchpad token tercepat berkembang di ekosistem Solana, di mana kreator mendapatkan 1% dari setiap trade. Namun, terdapat kesenjangan signifikan antara kapabilitas teknis platform dan pengalaman pengguna yang tersedia.

2.2 Pain Points yang Diidentifikasi
2.2.1 Untuk Token Creator
• Tidak ada tools untuk memproyeksikan pendapatan fee sharing berdasarkan volume historis dan antisipasi.
• Konfigurasi fee share bersifat manual dan membutuhkan pemahaman teknis mendalam tentang Solana PDA.
• Tidak ada visibilitas real-time atas accumulated fees yang siap di-claim.
• Proses launch token terpisah dari konfigurasi monetisasi, menyebabkan friction yang tinggi.

2.2.2 Untuk Trader & Investor
• Feed token launch di Bags.fm tidak menyertakan sinyal kualitas atau risk scoring.
• Tidak ada cara untuk membedakan token dengan potensi tinggi dari token rug-pull secara cepat.
• Data on-chain dari Helius dan price data dari Birdeye tersebar, tidak teraggregasi.
• Alert system untuk entry/exit points tidak tersedia secara native.

2.2.3 Untuk Developer & Partner
• Bags Partner API belum banyak dimanfaatkan karena kurangnya contoh implementasi nyata.
• Tidak ada dashboard untuk memantau performa referral dan partner revenue secara terpusat.

2.3 Opportunity Gap
Berdasarkan analisis terhadap Bags API documentation, tersedia 30+ endpoint yang belum dimanfaatkan secara maksimal oleh aplikasi manapun. Bags Signal hadir untuk menjadi aplikasi pertama yang mengintegrasikan seluruh surface area API Bags.fm menjadi satu platform kohesif.

5.2 Spesifikasi Fitur Detail
5.2.1 Signal Feed - Real-time Token Intelligence
Signal Feed adalah halaman utama platform yang menampilkan aliran token launch terbaru dari Bags.fm disertai dengan skor kualitas dari AI engine.

Komponen UI yang dibutuhkan:
• Token card dengan nama, symbol, gambar, dan status (PRE_LAUNCH / LIVE).
• Signal Score: badge berwarna (Hijau = 80+, Kuning = 50-79, Merah = <50).
• Volume 24 jam dari Birdeye API, diupdate setiap 30 detik.
• Liquidity depth dari Meteora pools.
• Quick action: Trade, Watch, Share.
• Filter: status, score range, volume minimum, tanggal launch.

Endpoint Bags API yang digunakan:
• GET /token-launch/feed - Feed utama token launches.
• GET /token-launch/lifetime-fees?tokenMint=X - Fee historis per token.
• GET /bags-pools - Informasi liquidity pool.

5.2.2 AI Scoring Engine
Scoring engine adalah layanan Python yang mengevaluasi setiap token secara otomatis menggunakan heuristik multi-faktor. Engine ini berjalan sebagai microservice terpisah yang dipanggil oleh backend Node.js.

Faktor penilaian (total 100 poin):
FaktorBobotDeskripsiLiquidity Depth25 poinTotal SOL dalam pool Meteora saat launch. > 50 SOL = skor penuh.Fee Share Config20 poinApakah creator telah mengkonfigurasi fee share dengan benar.Creator History20 poinTrack record creator: lifetime fees dari token sebelumnya.Volume Velocity15 poinRate kenaikan volume dalam 1 jam pertama setelah launch.Social Presence10 poinWebsite dan Twitter terisi, bukan placeholder.Price Stability10 poinVolatilitas harga dalam 1 jam pertama (rendah = lebih baik).
5.2.3 Fee Share Optimizer
Modul ini memungkinkan creator untuk mensimulasikan konfigurasi fee share terbaik berdasarkan target pendapatan dan volume trading yang diproyeksikan.

Input dari user:
• Estimasi volume trading harian (SOL atau USD).
• Jumlah wallet penerima fee (creator + partner).
• Target fee percentage yang diinginkan.

Output yang ditampilkan:
• Proyeksi pendapatan harian, mingguan, dan bulanan.
• Perbandingan konfigurasi: Current vs Optimal.
• Tombol "Apply Configuration" yang membuat transaksi Bags Fee Share Config langsung.
• Estimasi gas cost untuk transaksi konfigurasi.

Endpoint Bags API yang digunakan:
• GET /fee-share/wallet?walletAddress=X - Cek konfigurasi fee share existing.
• POST /fee-share/config - Buat transaksi konfigurasi fee share baru.
• GET /token-launch/claim-stats?tokenMint=X - Statistik klaim historis.

5.2.4 One-Click Token Launcher
Fitur flagship yang memungkinkan creator meluncurkan token, mengkonfigurasi fee share, dan mendaftarkan ke Dexscreener dalam satu alur terintegrasi.

Alur kerja (wizard 4 langkah):
1. Langkah 1: Token Info: Nama, symbol, deskripsi, upload gambar, link Twitter & website.
2. Langkah 2: Fee Configuration: Pilih persentase fee, tambah wallet penerima, preview pendapatan proyeksi.
3. Langkah 3: Launch Settings: Initial buy amount (opsional), harga awal, slippage tolerance.
4. Langkah 4: Review & Sign: Ringkasan semua setting, estimasi total gas, tombol "Launch Token".

Endpoint Bags API yang digunakan:
• POST /token-launch/info - Upload metadata token ke storage.
• POST /token-launch/transaction - Buat transaksi launch.
• POST /fee-share/config - Konfigurasi fee share bersamaan.
• POST /dexscreener/order - Auto-submit listing Dexscreener.

6.4 Smart Contract (Anchor Program)
Bags Signal mengembangkan Anchor program kustom dengan dua instruksi utama:

ClaimRouter - Auto Fee Claim
• PDA menyimpan konfigurasi: tokenMint, feeClaimerVault, minClaimThreshold.
• Instruksi claim_and_distribute: memanggil Bags fee claim lalu mendistribusikan ke multiple wallets secara atomik.
• Guard: hanya bisa dieksekusi jika accumulated fees > minClaimThreshold (mencegah dust claims).

PartnerRegistry - On-chain Partner Tracking
• Menyimpan mapping: partnerWallet -> [referredTokens] on-chain.
• Instruksi register_referral: mencatat setiap token yang dilaunched melalui Bags Signal.
• Memungkinkan partner revenue tracking yang transparan dan tidak bisa dimanipulasi.

9. Non-Functional Requirements
9.1 Performance
• Signal Feed latency: < 500ms dari request ke render pertama.
• AI Scoring latency: < 2 detik per token (cached setelah score pertama).
• WebSocket alert delivery: < 5 detik dari event on-chain.
• Trade Quote refresh: setiap 3 detik secara otomatis.
• API rate limiting: max 100 request/menit per wallet address.

9.2 Security
• API key Bags.fm disimpan hanya di server-side, tidak pernah di-expose ke frontend.
• Semua transaksi Solana ditandatangani client-side (private key tidak pernah meninggalkan wallet user).
• Input sanitization untuk semua form, mencegah XSS dan injection.
• HTTPS enforced di semua endpoint. CORS dikonfigurasi ketat hanya untuk domain produksi.
• Rate limiting berbasis IP untuk endpoint publik.

9.3 Reliability & Availability
• Target uptime: 99.5% selama periode hackathon dan demo.
• Fallback ke Helius RPC cadangan jika primary timeout.
• Redis cache memastikan Signal Feed tetap berfungsi meski Bags API momentarily down.
• Error boundary di semua React component - UI tidak blank saat partial failure.

9.4 Scalability
• Backend stateless, bisa di-scale horizontal dengan mudah di Railway.
• Redis digunakan untuk rate limiting dan session data - tidak bergantung pada in-memory state.
• Database query dioptimalkan dengan index pada tokenMint dan walletAddress.

9.5 Accessibility & UX
• Responsive design: full functionality di mobile (320px) hingga desktop (1920px).
• Dark mode dan light mode didukung secara native via Tailwind.
• Loading states pada semua async operations - tidak ada blank screen.
• Error messages bersifat actionable (bukan hanya "Something went wrong").
